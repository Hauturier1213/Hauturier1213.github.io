Hugo+Github部署个人博客

本文章主要介绍如何在Windows环境下用Hugo部署个人博客并用Github进行托管。

配置环境

  • 配置好Git环境
  • 有一个Github账号

配置好Git环境之后最好运行一下下列语句来使得git创建的仓库默认分支为main，以防后面出现问题

1

git config --global init.defaultBranch main

如果后续出现了本地分支为master而远程分支为main的话可以用以下语句来改变当前分支为main

1

git branch -M main

Hugo安装

  1. 打开Poweshell或者CMD运行以下内容安装Hugo：
1

winget install Hugo.Hugo.Extended
  1. 检查是否安装成功
    • 关闭当前打开的CMD窗口，重新开一个CMD窗口，输入：
1

hugo version

正确出现hugo相关信息则安装成功。

Github创建Blog仓库

  1. 打开Github，登陆后点击页面左上角的Github图标，然后创建新repository:

  1. 创建一个存放blog的私有仓库：

    • Repository name:仓库名，自行设置。

    • 其余配置按图进行选择，然后点击Create repository

  1. 进入刚创建好的仓库，点击绿色的Code按钮，然后复制链接：
  1. 克隆仓库到本地，选择一个文件夹（路径里最好不要有中文） 然后鼠标右键，点击Git Bash Here（如果你没有这个选项说明Git环境配置有问题，请另寻解决办法），然后在打开的窗口中输入：

git clone https://github.com/Hauturier1213/BlogSource.git#此处的链接是上一步中复制的链接

1

hugo new site try

出现如下信息说明成功

下载主题

在BlogSource文件夹中打开Git Bash，通过下面的语句来下载stak主题（也可以从Hugo主题社区选择自己喜欢的主题进行下载）

1

git submodule add https://github.com/CaiJimmy/hugo-theme-stack/ themes/hugo-theme-stack

若失败也可以用ssh进行下载（此方法需要进行额外的配置，请自行搜素Github SSH配置）：

1

git submodule add git@github.com:CaiJimmy/hugo-theme-stack/ themes/hugo-theme-stack

下载的主题会在themes文件夹中，然后将主题写入配置文件：

  • 此处的theme属性值是下载到themes文件夹中的主题文件夹的名字，一定要对应
1

echo "theme = 'hugo-theme-stack'" >> hugo.toml

将BlogSource\themes\hugo-theme-stack\exampleSite下的hugo.yaml文件复制到BlogSource文件夹中并删除原来的hugo.toml文件

然后将BlogSource\themes\hugo-theme-stack\exampleSite\content文件夹下的内容复制到BlogSource\content中

最后删除BlogSource\content\post中的rich-content文件夹

创建文章

使用以下语句来创建一个markdown文章：

1

hugo new post/test/index.md
  • 创建的Markdown文件路劲为/content/post/test/index.md
  • 根据路径找到index.md，打开写入一些内容并保存

注：

  • 创建的test文件夹就是一篇文章的目录，你可以将该文章中需要的图片等内容都放在test文件夹中
  • Markdown文件的名字一定要是index.md，否则后续hugo server后图片无法正常显示
  • 如果要创建新的文章，则类似上述的hugo语句，在post文件夹下另外创建一个文件夹

然后便可以预览创建的文章：

1

hugo server -D

根据给出的地址在浏览器中进行访问：

点击Test文章便可以看到文章内容：

部署到Github

创建Github Pages仓库

首先要先创建一个Github Pages仓库，该仓库必须使用特殊格式命名<username.github.io>，username是你的github用户名。

注意该仓库要设置为Public并且要勾选添加添加README文件选项。

name: deploy

# 代码提交到main分支时触发github action
on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
        - name: Checkout
          uses: actions/checkout@v4
          with:
              fetch-depth: 0

        - name: Setup Hugo
          uses: peaceiris/actions-hugo@v3
          with:
              hugo-version: "latest"
              extended: true

        - name: Build Web
          run: hugo -D

        - name: Deploy Web
          uses: peaceiris/actions-gh-pages@v4
          with:
          #只用改下面的两个地方
              PERSONAL_TOKEN: ${{ secrets.你的BlogSource的action secrets名字 }}
              EXTERNAL_REPOSITORY: 你的github用户名/你的github用户名.github.io
              PUBLISH_BRANCH: main
              PUBLISH_DIR: ./public
              commit_message: auto deploy
